# 贡献指南

感谢你愿意为「小杏仁」贡献力量！为确保协作顺畅、体验温柔，请遵循以下约定。

## 行为准则
- 温柔优先：尊重、倾听、友善交流
- 建设性反馈：问题聚焦在事情本身，避免人身评价
- 隐私保护：不得上传或泄露任何敏感信息与密钥

## 如何参与
- 提交 Issue：说明背景、复现步骤、期望行为、实际结果、环境信息
- 提交 PR：关联 Issue，描述变更动机与方案，列出关键修改点与影响范围

## 开发流程
1. Fork 仓库
2. 创建分支：`feat/xxx`、`fix/xxx`、`docs/xxx` 等
3. 提交信息：遵循 Conventional Commits（如 `feat: support memory anchors`）
4. 保持小步提交、可读性与可回滚性

## 代码规范
- 遵循项目现有代码风格与工具链
- 安全第一：不记录/打印秘钥，不上传私钥与令牌
- 不引入未使用的依赖，不随意添加体积过大的资源文件
- 新增代码类的 `author` 使用 `Ravey`（已有代码的 author 不修改）

## 文档规范
- 中文为默认语言；英文文档在 `README.en.md`
- 重要变更需更新 README 或相关模块文档

## 评审与合并
- 至少 1 位维护者审核通过
- 通过基础检查（Lint/格式/构建，如有）
- 变更范围清晰，风险可控，便于回滚

感谢你的耐心与善意，愿我们一起把「小杏仁」做成有温度、有智慧的成长伙伴。💚
